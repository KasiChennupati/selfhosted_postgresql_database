# Postgres Compliance and Legislation

Postgres is the most advanced and popular opensource relational database out there. 

So, when using this amazing database for storing your data based on the type of usecase we need to understand how can we put some compliance and audit controls so when we are using this database to store for business, enterprise usecases. we need to know if need to comply with any legulatory rules.

As a guiding list below policies and controls that are present globally applicable for for different usecases. 
We will not cover for what usecase the below applies only that if the below applies what processes, controls and mechanisms we need to put inplace so we are a safe, secure and compliant platform.

- GDPR/UK (UK) (General Data Protection Regulation for UK post brexit)
- EDPR/EU (European Union) (General Data Protection Regulation for EU)
- HIPAA (USA) (Health Insurance Portability and Accountability Act)
- HITECH Act (USA)
- CCPA / CPRA (USA)
- PIPEDA (Canada)
- Australian Privacy Act (Australia)
- New Zealand Privacy Act (New Zealand)
- DPDP Act (India)  (Digital Personal Data Protection Act)
- LGPD     (Brazil) (Brazils style of GDPR)
- PIPL     (China)  (Personal Information Protection Law)
- APPI     (Japan)  (Act on the Protection of the Personal Information)
- PIPA (South Korea) (Personal Information Protection Act)
- Data Bridge Privacy Shield (EU-US)
- ISO 27799:2016 (information security management in health domain)
